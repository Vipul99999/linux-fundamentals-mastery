# What is Open Source?

Open Source means:

> The source code of a software project is publicly available for anyone to view, study, modify, and distribute according to its license.

Example:

```text
Developer Creates Software
          ↓
Source Code Published
          ↓
Community Can Review
          ↓
Community Can Improve
          ↓
Software Evolves
```

---

# What is Source Code?

Source code is the human-readable instructions written by developers.

Example:

```c
#include <stdio.h>

int main() {
    printf("Hello Linux\n");
    return 0;
}
```

Humans understand source code.

Computers execute machine code.

# Closed Source Software

In closed-source software:

```text
Company
   ↓
Source Code
   ↓
Private
   ↓
Users Receive Binary Only
```

Examples:

* Microsoft Windows
* Adobe Photoshop
* Microsoft Office

Users cannot normally inspect the internal code.

# Open Source Software

In open-source software:

```text
Developer
    ↓
Source Code
    ↓
Public Repository
    ↓
Community Contributions
    ↓
Improved Software
```

Examples:

* Linux
* Git
* Docker
* Kubernetes
* PostgreSQL

# Why Open Source Exists

Before open source:

```text
Software
    ↓
Controlled By Companies
```

Problems:

* Expensive licensing
* Limited customization
* Vendor lock-in
* No transparency

Open source solved many of these issues.

# Open Source Licenses

Licenses define what users can do.

---

## GPL

Used by Linux.

Characteristics:

* Source code must remain open.
* Modifications must remain open.

---

## MIT License

Very permissive.

Allows:

* Commercial use
* Modification
* Redistribution

---

## Apache License

Common in enterprise software.

Adds:

* Patent protection

---

# Why Linux Uses GPL

Linux uses the GNU General Public License.

Purpose:

```text
Prevent Proprietary Ownership
```

If someone modifies Linux and distributes it:

```text
Modified Version
       ↓
Must Remain Open Source
```

This helped Linux stay open.
