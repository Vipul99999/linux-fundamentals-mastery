# Linux Signals — The Complete Deep Dive

> Signals are one of the oldest and most important Inter-Process Communication (IPC) mechanisms in UNIX and Linux.
>
> They are used everywhere:
>
> - Linux Kernel
> - Bash
> - systemd
> - Docker
> - Kubernetes
> - Databases
> - Web Servers
> - Monitoring Systems
> - Programming Languages
>
> If processes are the "workers" of Linux,
>
> then signals are the "messages" used to control those workers.

# What is a Signal?

A signal is:

```text
An asynchronous notification
sent to a process.
```

In simpler words:

```text
A message sent to a process.
```

---

# Most Important Signals

---

## SIGHUP (1)

Meaning:

```text
Hang Up
```

Modern use:

```text
Reload Configuration
```

Example:

```bash
kill -HUP nginx_pid
```

---

## SIGINT (2)

Generated by:

```text
CTRL+C
```

Purpose:

```text
Interrupt Process
```

---

## SIGQUIT (3)

Generated by:

```text
CTRL+\
```

Action:

```text
Terminate
+
Core Dump
```

---

## SIGTERM (15)

Graceful shutdown request.

Visual:

```text
Please Exit Cleanly
```

---

## SIGKILL (9)

Immediate termination.

Visual:

```text
Die Now
```

Cannot be:

```text
Ignored
Caught
Blocked
```

---

## SIGSTOP (19)

Pause process.

Cannot be ignored.

---

## SIGCONT (18)

Resume paused process.

---

## SIGCHLD

Sent to parent when:

```text
Child State Changes
```

Example:

```text
Child Exits
```

---

## SIGSEGV

Segmentation fault.

Visual:

```text
Invalid Memory Access
```

---

## SIGBUS

Memory access error.

Often hardware-related.

---

## SIGFPE

Arithmetic error.

Example:

```text
Divide By Zero
```

# Signal Handlers

Programs can intercept signals.

Example:

```c
void handler(int sig)
{
    printf("Signal received");
}
```

---

# Why Useful?

Example:

```text
Web Server
```

receives:

```text
SIGTERM
```

Instead of dying immediately:

```text
Stop New Requests
Finish Existing Requests
Close Connections
Exit
```

---

# signal() vs sigaction()

Two APIs exist.

---

## signal()

Older.

```c
signal(SIGINT, handler);
```

---

## sigaction()

Modern.

Preferred.

```c
sigaction(...)
```

More reliable and powerful.

---

# Why sigaction()?

Supports:

```text
Signal Masks

Flags

Advanced Handling
```

Used in production systems.

# Why Block Signals?

Prevent interruptions during:

```text
Critical Operations
```

Example:

```text
Updating Shared Data
```

---

# Pending Signals

```text

```

Visual:

```text
Signal Sent
      │

```

# Once Unblocked

```text
   │
   ▼
Delivered
```

# Real-Time Signals

Linux provides:

```text
SIGRTMIN
...
SIGRTMAX
```

---

# Why Real-Time Signals?

Standard signals:

```text
May Merge
```

Real-time signals:

```text
Queued
Ordered
Reliable
```

---

# Standard Signal Problem

Multiple:

```text
SIGUSR1
```

may become:

```text
One Pending Signal
```

---

# Real-Time Signal Solution

Each signal preserved.

Visual:

```text
Signal 1

Signal 2

Signal 3

All Delivered
```

---

# Signal Queues

Real-time signals support queues.

```text
Signal A

Signal B

Signal C
```

Processed in order.

---
