# Virtual File Systems (Virtual Filesystems) in Linux

> One of the most powerful ideas in Linux is:
>
> ```text
> Everything is a File
> ```
>
> But how can CPU information, memory statistics, running processes, USB devices, network interfaces, and kernel settings be files?
>
> The answer is:
>
> **Virtual File Systems (VFS-based pseudo filesystems)**.
>
> Virtual filesystems expose kernel and system information through files and directories even though no real data exists on disk.

# What is a Virtual Filesystem?

A Virtual Filesystem is a filesystem whose files are generated dynamically by the kernel.

Unlike traditional filesystems:

```text
ext4
xfs
btrfs
```

virtual filesystems usually:

```text
Do Not Store Data On Disk
```

# Real Filesystem vs Virtual Filesystem

| Feature | Real FS | Virtual FS |
|----------|----------|----------|
| Stored on Disk | Yes | Usually No |
| Persistent | Yes | Usually No |
| Uses Blocks | Yes | No |
| Dynamic Data | Rare | Yes |
| Generated by Kernel | No | Yes |
